How the review pipeline works

Every push runs in an isolated worktree, never your live working directory, so validation never disrupts what you're doing locally. 

flowchart TD
    A["Your branch"] --> B["Disposable worktree
(isolated copy, your files untouched)"]
    B --> C["Review"] --> D["Test"] --> E["Docs"] --> F["Lint"]
    F --> G{"All checks passed?"}
    G -- "yes" --> H["Push to origin"]
    H --> I["Open PR"]
    I --> J["CI"]
    G -- "no, and not auto-fixable" --> K["Finding surfaced
for your decision"]
    G -- "no, but safe to auto-fix" --> L["Fix applied automatically"] --> C

The pipeline distinguishes safe, automatic fixes from policy decisions that need a human — only the latter block you.

The “gate”: a second remote in front of origin

no-mistakes init doesn't touch your existing remote — it adds a new one that intercepts pushes first. 

flowchart LR
    L["Local clone
project_template"] -- "git push no-mistakes branch" --> G[("Bare gate repo
~/.no-mistakes/repos/<id>.git")]
    G -- "post-receive hook
triggers daemon" --> D["no-mistakes daemon"]
    D -- "pipeline passes" --> O[("origin
github.com/gmei6/project_template")]
    O --> P["Pull Request opened"]
    L -. "git push origin branch
(unchanged, bypasses the gate)" .-> O

How to run it locally

Already initialized in this repo — these are the day-to-day commands.

  • Make changes on a branch
  • Push through the gate
  • Watch or review findings
  • PR opens on origin

Run the gate

git push no-mistakes my-branch

Runs review → test → docs → lint in the disposable worktree, then pushes to origin and opens a PR if everything passes.

Watch it live / review by hand

no-mistakes attach
no-mistakes

attach follows the active run; the bare command opens the interactive TUI for manual review.

Check state

no-mistakes status
no-mistakes runs
no-mistakes doctor

Gate/daemon status, pipeline run history, and a health check of git/gh/agents.

Retry or undo

no-mistakes rerun
no-mistakes eject

rerun re-runs the pipeline for the current branch; eject cleanly removes the gate from this repo (your normal origin push keeps working either way).

Regular git push origin <branch> still works unchanged and completely bypasses the gate — no-mistakes only runs when you explicitly push to the no-mistakes remote.
